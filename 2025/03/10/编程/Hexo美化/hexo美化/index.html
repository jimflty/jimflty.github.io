Hexo学习笔记

2025/3/10 编程 Hexo

Hexo建站完毕，还需经过一定的美化达到观感上的满意。Hexo可进行美化的部分包括主题、背景，以及文档排版（layout）。Hexo初始主题已经非常简洁美观，仅需对背景、文档排版等进行美化即可。本文用于记录美化过程中发现和排除的问题，查看最终美化结果。

自定义背景

根据Hexo 自定义本地封面图中提到的方法：

  1. 图片的http链接

  2. 把图片放在博客根目录/source/images

  3. 通过 abbrlink 和 assetimg 的方式（需要安装abbrlink&&asset-img）

三种方式均可实现自定义背景，但考虑到http有内容替换的可能性，把心仪的图片保存到本地的方式更为稳妥且不用额外安装插件，因此选择采用2。

由于距该教程已过去两年，Hexo文件结构已然发生变化，现在放置背景图片的目标文件夹为根目录/public/css/images

我们将图片放置到目标文件夹并命名为banner.jpg，即可完成背景替换。

文档排版

layout

layout有助于更好地控制博客页面的结构和样式，使我们能够根据需求轻松创建和管理不同类型的页面和文章。详细教程：Layout详解

layout在官网文档中的另一种说法是布局，默认有三种：

  1. post（页面仍然为Hexo主题）
  2. page（新建页面）
  3. drafts（草稿，不会在blog中显示，后续可通过publish命令发布为post）

Front-matter

Front-matter用于post在博客页面的显示和分类，常用属性有：

设置 描述 默认值
title 标题 文章的文件名
date 建立日期 文件建立日期
tags 标签
categories 分类
permalink 覆盖文章的永久链接. 永久链接应该以 / 或 .html 结尾 null
excerpt 设置文章在博客页面显示的摘要，默认显示全文

在这些属性中excerpt对美化帮助最大，显示摘要极大程度上减少了博客页面的冗余。

主题

2025.7.5 刚从大佬的blog回来，看了别人的美化向来喜欢极简的我心血来潮想尝试新主题，换换口味。

本次将要更换的主题是ParticleX主题介绍，相比默认该主题更强调个性化，且突出显示的个人介绍界面也非常讨喜。

好在Hexo主题安装非常方便，只需将主题克隆到themes文件夹，再在config文件中替换即可。接下来针对两个主题默认设置上的差异进行一些调和与个性化设置，便可完成主题适配。

更换主题

cd themes
git clone https://github.com/theme-particlex/hexo-theme-particlex.git particlex --depth=1

在_config.yml中定位themes，将默认内容更改为particlex，完成更换。

调整设置

关闭代码高亮
Hexo 有自带的代码高亮，但是和 ParticleX 的不兼容。需要删除配置文件中syntax highlight:后面的内容。

随机背景
background参数是一个列表，打开时会随机加载一个背景。

# Avatar image
avatar: /images/avatar.jpg

# Home page background image
background:
    - /images/background.jpg

# Loading image
loading: /images/loading.gif

# Optional colors for category and tag
colors:
    - "#ffa2c4"
    - "#00bcd4"
    - "#03a9f4"
    - "#00a596"
    - "#ff7d73"

主页信息卡片
description 支持 Markdown 格式。

图标链接 iconLinks 配置和导航栏配置相同。

# Side info card
card:
    enable: true
    description: |
        Description
        ...
    iconLinks:
    friendLinks:

页脚
目前网站还是挂载在Github，若以后使用个人域名，需在页脚添加备案信息。

# Footer info
footer:
    since: 2022
    # Customize the server domain name ICP
    ICP:
        enable: false
        code:
        link:

文章置顶
在 Front-Matter 里设置 pinned 作为置顶参数，越大越靠前，默认为 0。
